## 本轮优化方向

### Phase 6: 数据关联与商业化完善

#### Task 1: 用户数据关联
- **问题**: 所有用户数据存储在相同的 localStorage key 中，多用户共用设备时数据混乱
- **方案**: 所有 localStorage key 添加当前用户 ID 前缀，未登录用户使用 `guest` 前缀
- **修改文件**:
  - `cart-provider.tsx`: `xmstore-cart-${userId}`
  - `wishlist-provider.tsx`: `xmstore-wishlist-${userId}`
  - `order-provider.tsx`: `xmstore-orders-${userId}`

#### Task 2: 订单状态管理
- **新增** `OrderStatus` 类型: `pending` | `shipped` | `delivered` | `cancelled`
- **Order 接口扩展**: 添加 `status`, `shippedAt`, `deliveredAt` 字段
- **新增方法** `updateOrderStatus`: 支持状态流转并自动记录时间戳
- **订单列表页** (`orders/page.tsx`): 使用 Badge 组件展示状态，支持颜色区分
- **订单详情页** (`orders/[id]/page.tsx`): 展示完整状态信息和描述

#### Task 3: 用户地址管理
- **新建** `address-provider.tsx`: Address Context，支持增删改查和默认地址设置
- **新建** `profile/addresses/page.tsx`: 地址管理页面，支持表单添加、删除、设为默认
- **数据隔离**: `xmstore-addresses-${userId}`

#### Task 4: 优惠券系统
- **新建** `coupon-provider.tsx`: Coupon Context，支持验证和使用优惠券
- **默认优惠券**: `WELCOME` (满200减50), `SAVE100` (满500减100)
- **验证逻辑**: 检查 code、未使用、未过期、满足最低金额
- **数据隔离**: `xmstore-coupons-${userId}`

#### Task 5: PWA 图标
- **新建** `public/icons/icon.svg`: XM Store 品牌 SVG 图标
- **更新** `manifest.json`: 使用 SVG 图标，支持 `any` 尺寸和 `maskable`

#### Task 6: 订单详情页
- **新建** `orders/[id]/page.tsx`: 展示订单状态、时间线、商品清单、金额明细、收货信息
- **空状态处理**: 订单不存在时展示友好提示

#### Task 7: 用户个人中心
- **新建** `profile/page.tsx`: 用户信息卡片、统计面板（订单/地址/收藏）、快捷菜单
- **未登录状态**: 引导用户登录

#### Navbar 更新
- **桌面端**: 用户名链接到个人中心，添加 Settings 图标
- **移动端**: 菜单中添加"个人中心"入口

#### Layout 更新
- **Provider 嵌套**: `UserProvider > AddressProvider > CouponProvider > CartProvider > WishlistProvider > OrderProvider`
